Prerequisites
Before applying anything, confirm these are already running on your cluster:

ArgoCD installed in the argocd namespace
NGINX Ingress Controller
cert-manager with a selfsigned-issuer ClusterIssuer

If cert-manager or the issuer aren't set up yet, the ingress TLS will fail silently — worth verifying first.

Step 1 — Create the files
Create the three files exactly as shown in your structure:
gitops/
├── root-app.yaml
└── apps/
    └── argo-workflows/
        ├── application.yaml
        └── ingress.yaml
Copy the YAML content you have above into each file.

Step 2 — Commit and push
powershellgit add .
git commit -m "feat: add argo-workflows app of apps gitops structure"
git push origin main

Step 3 — Connect your repo to ArgoCD
In the ArgoCD UI: Settings → Repositories → Connect Repo

Type: HTTPS
URL: https://github.com/rgtdev95/kubernetes-journey
If the repo is private, add your GitHub credentials or a personal access token

Click Connect and confirm it shows a green checkmark.

Step 4 — Apply the root-app (one time only)
bashkubectl apply -f gitops/root-app.yaml
That's the only manual kubectl apply you'll ever run. ArgoCD takes it from here — it will discover gitops/apps/argo-workflows/application.yaml automatically and deploy everything.

Step 5 — Access the UI
Add this to your /etc/hosts (or C:\Windows\System32\drivers\etc\hosts on Windows):
<your-node-ip>   argo-workflows.local
Then open https://argo-workflows.local in your browser. Since it's a self-signed cert, you'll need to accept the browser warning.

Troubleshooting tips
If the sync fails in ArgoCD, the most common causes on kubeadm clusters are:

The argoworkflows namespace not being created (the CreateNamespace=true sync option handles this, but check ArgoCD logs if it errors)
cert-manager's selfsigned-issuer not existing — create it first if needed
The ServerSideApply=true option requiring a relatively recent Kubernetes version (1.22+)
